麻雀の学習アプリを作りました。といっても本当の狙いは別にあって、「AIにキャラの絵を描かせる練習がしたかった」が動機の半分を占めています。その過程で得た画像生成のノウハウを、アプリの紹介とあわせて残しておきます。



## 作ったもの：Mahjo

麻雀の役と点数の数え方を学習するためのアプリです。ブラウザだけで遊べます。

https://mahjo.academy/

ランダムな和了（あがり）形を出題し、「どんな役が成立するか」「点数をどう数えるか」をビジュアルで見せて学びます。役モードと点数モードがあり、答えをすぐ教えない段階ヒントと、サポートキャラが学習に付き添ってくれるのが特徴です。

## なぜ作ったか

いずれゲームを作りたいと思っています。そしてゲームには絵が要る。ところがAIの画像生成は、「それっぽい1枚」を出すのは簡単でも、「同じキャラを、何枚も、狙った構図とポーズで」揃えるのが途端に難しくなります。立ち絵があって、表情差分があって、サムネがあって……と量産しようとすると、顔も衣装の柄もどんどんブレていく。

この「同一キャラを安定して量産する」練習にちょうど良かったのが、麻雀学習アプリでした。対局（実戦の打牌・進行）は作らないと割り切れば、ゲームとしてのスコープはぐっと狭くなる。その代わりサポートキャラという形で、立ち絵・表情差分・アバターを作り込む口実ができる。麻雀の勉強アプリの皮をかぶった、AI作画の練習場というわけです。

![まおの立ち絵](/images/ai_product_mahjo/mao-full.webp =250x)


## 画像生成は nano banana 2 を Gemini アプリ経由で

画像生成には nano banana（Google の画像生成モデルの通称）を使いました。正確には Gemini アプリ経由で。

理由は単純で、Gemini アプリなら無料枠で最新の nano banana 2 が触れるからです。nano banana 2 は2026年2月に発表されたモデルで、正式名称は `Gemini 3.1 Flash Image`。Gemini アプリの無料ユーザーでも1日20枚ほど生成できます（上限は頻繁に変わるとのこと）。

https://blog.google/innovation-and-ai/technology/ai/nano-banana-2/

注意点として、コンシューマ版の Gemini アプリは使うモデルを明示的に指定できないので、ある生成で本当に nano banana 2 が当たったかの保証はありません。きっちりモデルを固定したいなら AI Studio や Gemini API という開発者向けの経路がありますが、最新の nano banana 2 は基本有料。「とにかく枚数を回して試行錯誤する」という練習目的では、お金をかけずに最新モデルを叩ける Gemini アプリ経由が都合よかったです。

なお AI Studio でも、ひとつ前の nano banana 1（初代）なら無料枠で使えます。モデルを固定できる安心感はありますが、絵の質は nano banana 2 のほうが明らかに上でした。2 のほうが線も塗りも安定して、何より活き活きとしたキャラクターが描ける。表情やポーズに生気が乗る感じで、キャラを作り込むなら 2 を使いたい、というのが正直な感想です。だからこそ、モデルを固定できないリスクを承知のうえで Gemini アプリ経由（nano banana 2）を主軸にしました。



## 同じキャラを安定して出す3つのコツ

同一キャラを量産しようとして詰まった点と、その対処を3つ。


### コツ1：ポーズ違いは i2i でなく同一セッションの t2i で

画像生成には大きく2つのやり方があります。

- t2i（text-to-image）: テキストプロンプトだけから新規に描く。線がくっきり出て画質が高いが、顔や衣装の柄がブレやすい。
- i2i（image-to-image）: 元画像を渡して「これを別ポーズで」と編集させる。一貫性は保ちやすいが、重ねるほど線が眠くなる（世代劣化）。

最初は素直に「masterの絵を1枚作って、それを i2i で塗り替えてポーズ違いや表情差分を作ればいい」と考えました。ところが i2i は1回ごとに画像全体を再生成するので、模様替え→ポーズ→鮮明化……と重ねるたびに輪郭が甘くなっていきます。透過（背景抜き）の綺麗さも元絵の品質が上限なので、もやっとした絵からは白フチやギザつきが取れません。

結局たどり着いた最善手は、masterを生成したのと同じセッション（同じ会話）を維持したまま、t2i で「同じキャラを別ポーズで」と頼むことでした。同一セッションなら文脈でキャラの一貫性が保たれ、かつ t2i なので線はくっきりのまま。i2i の世代劣化を避けられます。実際、別セッションで master から i2i すると輪郭が荒れたものが、master を作ったセッション内で t2i し直すと一貫性を保ったまま鮮明になりました。

ポーズ違い・立ち絵バリエーションは、可能なら master と同じセッションで t2i する。i2i は「ごく小さな表情差分だけ」の局所手段に絞る、というのが結論です。


### コツ2：master さえあればセッションは作り直せる

「同一セッションで回す」が最善とはいえ、セッションは切れます。長く回せば品質も濁ってくる。

ここで効いたのが、master 画像さえ残っていればセッションは作り直せるという発見でした。新しいセッションの冒頭で master 画像を添付し、「これがこのキャラの正準リファレンス。以降この設計を厳密に維持して」と宣言したうえで、固定したい要素（顔の造形・髪・髪飾り・衣装・配色・画風）を箇条書きで並べる。そして肝心なのが「塗り替え（i2i）ではなく、毎回この画風で新規に t2i して」と明示することです。

実際に使っている宣言テンプレートはこんな形です。

```text
This is my original character "<name>". The attached image is the CANONICAL
REFERENCE for her. Keep her design EXACTLY consistent with it for everything
that follows.

Lock these traits (do not redesign):
- <顔の造形・瞳, 髪+髪飾り, 衣装, 配色, モチーフ>
- art style: clean crisp anime lineart, sharp clean outlines, smooth cel shading

How to draw from now on:
- Draw EACH request as a FRESH, CLEAN illustration in this exact style. Do NOT
  just repaint or blur the reference — redraw cleanly so the lines stay crisp.
- Always: single character, plain solid white background, no watermark, no text.

Confirm you've locked <name> as the reference, then wait for my next instruction.
```

これで新セッションでも元と同じ一貫性＋鮮明さで描けます。再構築したセッションで出した表情差分が、master と衣装の模様まで一致したときは、ちょっと感動しました。「masterを作ったセッションを大事に使い回す」のがベストですが、切れても致命傷ではない、と分かったのが大きい。


### コツ3：柄はゾーンに固定して「指示外は足すな」

衣装に模様を入れると、これが生成のたびにバラけます。最初は「星を散りばめた衣装」にしていたのですが、星撒きのようなランダムな全面配置は生成ごとに位置が変わって再現性ゼロ。同じキャラのはずなのに、絵によって柄がまったく違う、という事故が多発しました。

効いたのは、各モチーフを体のゾーン（肩・胸・腰・腿・裾など）に固定して指定するやり方です。コツがいくつかあります。

- 形と位置を具体的に書く。「肩に1つ、腰に1つ」のように数と場所を固定する。
- 実在の星座名で指定する。モデルが形を学習済みなので安定し、向きも指せる（例：カシオペアを横向きのWの形で）。星はその星座の節点だけに絞り、ばら撒かない。
- 「指示外の柄は足すな」を明記する。放置するとモデルが余分な星座や模様を勝手に盛って、画像間でズレます。`use ONLY the listed motifs — do NOT add any unspecified pattern` の一文が、再現性にいちばん効きました。

要するに、模様を「閉じた集合」としてゾーンに割り付けてしまう。これだけで、別ポーズの絵を出しても柄が揃うようになりました。



## 仕上げは透過とWebP化

生成した絵は、背景を透過してWebPに変換してから組み込みます。加工はブラウザ内で完結するツールを使いました。画像をサーバーに送らずローカルで処理できるので、プライバシー面でも安心です。

- Photopea … 白背景を透過にする、切り出す、PNGで書き出す。
- Squoosh … 表示サイズにリサイズしてWebPで書き出す（透過維持）。

ひとつだけ順序の注意点。透過してからリサイズすること。白背景のままリサイズすると、縁が白と混ざって白フチ（フリンジ）が出て、あとから取れなくなります。


### ImageMagick なら Claude Code から加工できる

手作業のGUIだけでなく、ImageMagick を使えば Claude Code（CLI）からそのまま加工できます。これが思いのほか快適でした。

```bash
# 透過済みPNGをWebP化
magick in.png -strip -define webp:method=6 -quality 90 out.webp

# アバター用に1:1へクロップ→リサイズ（リサイズは必ず透過後）
magick in.png -resize 192x192 avatar.webp
```

何が良いかというと、出力された画像を見ながら次の指示が出せる点です。「もう少し上を詰めて」「このサイズだと顔が小さいからクロップ範囲を変えて」といった調整を、コマンドの数値をいじりながら対話的に回せる。GUIでポチポチやるより、トリム位置やクロップ範囲を数値で再現しながら詰められるので、複数キャラ・複数差分を同じ枠で揃える作業と相性が良かったです。

## まとめ

麻雀学習アプリの皮をかぶせて、AI作画の練習をしてきました。得られた手応えをまとめると、

- 同じキャラの量産は t2i × 同一セッションが基本。i2i の重ねがけは線が眠くなる。
- セッションは master さえあれば再構築できる。宣言テンプレで一貫性を引き継ぐ。
- 柄はゾーンに固定して「指示外は足すな」。これで再現性が出る。
- 加工は ImageMagick で Claude Code から対話的に回すと速い。
- モデルは Gemini アプリ経由の nano banana 2 が、無料で最新を試せて練習向き。

「同じキャラを安定して出す」感覚がだいぶ掴めたので、次は実際に動きのあるゲームに挑戦してみたいと思います。

……と、ここまで「練習のための小さなアプリ」というつもりで書いてきたのですが、白状すると、世界観を作り込んでいくうちにキャラへの愛着が湧いてしまいました。気づけばドメインまで取って公開してしまっています。当初の「ちょっとした練習台」という位置づけはどこへやら。

まだ実装できていないキャラもたくさんいるので、これからも少しずつ追加していきたいと思っています。練習から始まったアプリですが、もうしばらく育ててみるつもりです。
