最後の更新から気づけばだいぶ時間が空いてしまいました。
サボっていた理由は単純で、AI開発が楽しすぎたからです。

Claude Codeを使ってWebアプリを作り始めたら、これが想像以上に面白くて、記事を書く暇があったらコード書いてました。
マンガやゲームなどをやる時間も、全部AIに吸われましたね。


## 作ったもの：tripcraft.fun

旅行の計画をみんなで楽しく作るWebアプリを作りました。

👉 [tripcraft.fun](https://tripcraft.fun)

![tripcraft.funのトップページ](/images/ai_product_tripcraft/top.png)

「やりたいこと」を出し合って、それを旅程に組み立てていく、というコンセプトです。
ありがちな「決まった予定を入力するだけ」のアプリではなく、「計画を立てる過程そのものを楽しむ」ためのアプリです。


### 主な機能

- 旅のプランを作成して、行先・日程などの基本情報を設定
- 招待リンクで仲間を招待（ゲスト参加もOK、ユーザー登録不要）
- 「やりたいこと」を追加して、仲間とリアクションやコメントで盛り上がる
- やりたいことを盛り込んだ旅程を作成
- ログインなしでも使える（データはlocalStorageに保存）
- オフラインも行けるはず（テスト甘いかも）

![やりたいことの一覧](/images/ai_product_tripcraft/wish_items.png)
![旅程](/images/ai_product_tripcraft/itinerary.png)


### 技術スタック

- **フロントエンド**: React + TypeScript + Tailwind CSS
- **ルーティング**: TanStack Router（ファイルベース）
- **バックエンド**: Firebase Functions
- **DB**: Cloud Firestore
- **認証**: Firebase Authentication
- **ホスティング**: Firebase Hosting

いわゆるFirebaseにフルベットした構成です。ログインなしでも使えるようにlocalStorageとFirestoreのデュアルライト方式にしていたり、招待リンクでのゲスト参加にカスタムトークン認証を使っていたり、個人開発のわりにけっこう凝った設計になってます。


## Claude Codeでの開発スタイル

### vibeコーディング？

いわゆる「vibeコーディング」に含まれる開発スタイルだと思うのですが、当初思ってたより細かく指示を出しましたね。後ろに数字を載せます。
最終的な１トピックのAI開発の流れは以下のスタイルに落ち着いています。

1. 設計、方針、データ構造について指示してAIに提案させる
2. 納得いくまでラリーを繰り返して決定する
3. ドキュメントやデータ構造のみを実装させる
4. 処理の実装計画を提案させる
5. 納得いくまでラリーを繰り返して決定する
6. 処理を実装させる
7. 出来上がったものの動作確認
8. 触って気になったコトを修正or別バックログに積み増し
9. 完了

これって、よくあるアジャイル開発の姿だなーって印象です。ただ、スピード感が１０倍ぐらいなのですけどね。よく言われることですが、AIは開発を加速させる。をまさに実感しましたね。

### AIの実力

やりとりしていた感覚として、AIの提案は2〜3割くらいの確率で保守性を下げる提案をしてきます。
保守性の勘所が若手技術者レベルって感覚がぴったりきます。

特に、ドキュメント、データ構造などはイマイチだった気がします。
逆に、処理を書かせるときちんとしたコードが出てくることが多かったです。
なので、方針やデータ構造についてAIと細かくラリーして決めてから、開発をやらせるというスタイルになりました。


### ドキュメント駆動が効く

このプロジェクトで一番うまくいったと思うのは、ドキュメントを先に書いてからAIに実装させるというやり方です。

CLAUDE.mdに加えて、10ファイル程度のドキュメントを整備しました。アーキテクチャー、データモデル、機能仕様、デザインガイドライン、テスト方針…。これらをAIが参照することで、プロジェクト全体の方針と一貫した実装が出てくる確率がかなり上がりました。

また、実装しているとドキュメントと実装の乖離が出てくるので、ドキュメントもちゃんとリファクタリングしていく必要があります。


### バックログ管理は人間の仕事

バックログ管理のコミットが100回というのは、自分でも多いなと思います。でもこれが大事でした。

AIは「今の実装タスク」には強いけど、「次に何をやるべきか」「全体の優先順位はどうか」という判断は人間がやる必要があります。機能を1つ実装するたびにバックログを見直して、新しく見つかった課題を追加して、優先度を整理して…という作業を愚直にやっていました。


## まとめ

Claude Codeを使った開発は本当に楽しかった。でも「AIに任せれば全部やってくれる」わけではなくて、むしろ設計とタスク管理を人間がしっかりやることで、AIの力を引き出せるという実感です。

tripcraft.funはまだまだ発展途上で、AI機能の搭載など、やりたいことがたくさんあります。
また更新が止まったら、きっと何か作ってるんだと思ってください。
